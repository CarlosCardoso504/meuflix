Sobre o MeuFlix

O MeuFlix é uma plataforma onde qualquer pessoa cria sua própria página de vídeos estilo Netflix, com fotos, título e vídeos que ela mesma sobe. Cada página é privada e isolada das demais: o link secreto de edição é gerado automaticamente e é o que garante que só o criador da página consiga adicionar ou remover vídeos, dois links são gerados um para o editor e um para quem receber o link não editavel, ideal para presentes de aniversário, namoro, dia dos pais etc...

Tecnologias usadas

  • Front-end: HTML5, CSS3 e JavaScript puro (sem frameworks), com layout responsivo inspirado no estilo Netflix.
  • Back-end: Node.js com o framework Express.js, responsável pela API REST que cria páginas e cadastra/lista/remove vídeos.
  • Banco de dados: PostgreSQL, com duas tabelas relacionadas: pages (cada página criada, com um código público e uma chave secreta de edição) e videos (cada vídeo, vinculado à sua página).
  • Upload de arquivos: biblioteca Multer, que recebe a imagem de capa e o vídeo enviados pelo formulário.
  • Segurança: módulo nativo crypto do Node.js, usado para gerar os códigos aleatórios que identificam cada página e sua chave secreta de edição.

Como foi feito

O projeto foi construído em três camadas. A camada de apresentação é um site estático (HTML/CSS/JS) que consome uma API. A camada de aplicação é um servidor Node.js/Express que expõe rotas para criar páginas (POST /api/pages), listar os vídeos de uma página (GET /api/pages/:slug), adicionar um vídeo (POST /api/pages/:slug/videos) e remover um vídeo (DELETE /api/pages/:slug/videos/:id). As rotas de adicionar e remover exigem uma chave secreta (edit_token) enviada num cabeçalho da requisição — sem ela, o servidor recusa a ação. A camada de dados é um banco PostgreSQL, onde cada página vira uma linha na tabela pages e cada vídeo uma linha na tabela videos, vinculada à sua página. Todo o código foi versionado e implantado em um provedor de nuvem, conforme detalhado abaixo.

Infraestrutura utilizada

Para colocar o sistema no ar, simulando a infraestrutura de uma empresa real, foram usados os seguintes componentes:

Servidor

Web Service Linux (container Node.js) hospedado na Render, responsável por rodar a aplicação Express e servir a API e os arquivos estáticos.

DNS

Gerenciado pela Render através do domínio padrão meuflix.onrender.com, com a opção de apontar um domínio próprio via Cloudflare DNS.

Serviço Web

Node.js + Express.js, expondo uma API REST (JSON) e servindo o front-end estático na mesma aplicação.

Banco de Dados

PostgreSQL gerenciado (Render Postgres), armazenando as páginas criadas e os vídeos de cada uma delas.

Firewall

Proteção de borda via Cloudflare (WAF e mitigação de DDoS) somada ao firewall de rede interno da própria Render, que isola o serviço da internet pública.

Nuvem

Render.com, uma plataforma de nuvem (PaaS) que executa a aplicação sobre infraestrutura da AWS, oferecendo deploy automático a partir do GitHub.
